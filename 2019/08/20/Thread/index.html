多线程的创建方式

1.继承Thread类

2.实现Runnable接口

3.实现callable接口

多线程创建的三种方式的对比

继承thread类

优点：

可以直接使用Thread类中的方法，代码简单

缺点：

继承Thread类之后就不能继承其它的类

实现runnable接口

优点：

即使自定义类有了父类，也不影响，因为可以实现多个接口。

缺点：

在run方法内部需要获取到当前线程的Thread对象，才能使用Thread类的方法

实现callable接口

优点：

可以获取返回值，可以抛出异常

缺点：

代码编写较为复杂

线程的生命周期

新建，就绪，运行，阻塞，死亡

1.新建线程，创建一个线程对象。

2.线程对象创建好之后进入就绪状态，此时会等待获取CPU的执行权。

3.获取到CPU执行权之后，线程对象开始运行。

4.在线程对象运行的过程中，可能会有CPU会切换到其他线程上面，此时会失去执行权，回到第二步就绪状态。

5.如果线程对象在执行过程中遇到了wait()方法或者sleep()方法，线程将会进入阻塞状态。

6.线程对象执行结束后会变成死亡状态。

生产者消费者设计模式

在一个KFC内。服务员负责生产食，消费者负责消费食物。

涉及到的内容：

1.Java模式思想：生产着消费者模式

2.要保证操作对象的一致性，即消费者和服务者都是跟同一个KFC发生关系的，KFC只能new一次。

3.this.notifyAll();和this.wat();

唤醒所有线程；线程等待；

如：生产者生产完毕，唤醒所有线程（生产者和消费者），再让自己（生产者）等待；

消费者开始消费，直到所有食材不够，先唤醒所有进程（包括生产者和消费者），在让自己（消费者）等待。

如此循环下去。

4.生产者和消费者都要继承Thread类，才能实现线程启动。

程序设计思路：

1.创建一个食物类，有存放/获取食物的方法

2.创建一个KFC类，有生产食物和消费食物的方法

3.创建一个客户类Customer，继承Thread，重写run()方法，在run()方法里面进行消费食物操作

4.创建一个服务员类Waiter，继承Thread类，重写run()方法，在run()方法里面进行生产食物的操作

5.创建主方法的调用类

线程池

newCachedThreadPool

创建一个可缓冲的线程池

工作线程的数量几乎没有限制（其实有限制的，数目为Integer.MAX_VALUE）

public static ExecutorService newCachedThreadPool()
创建一个可缓存的线程池，调用execute将重用以前构造的线程（如果线程可用）。如果现有线程没有可用的，则创建一个新线 程并添加到池中。终止并从缓存中移除那些已有 60 秒钟未被使用的线程。

newFixedThreadPool

public static ExecutorService newFixedThreadPool(int nThreads)
创建固定数目线程的线程池。

newSingleThreadExecutor

public static ExecutorService newSingleThreadExecutor()

创建一个单线程化的Executor，即只创建唯一的工作者线程来执行任务。

newScheduleThreadPool

创建一个定长的线程池

public static ScheduledExecutorService newScheduledThreadPool(int corePoolSize)
创建一个支持定时及周期性的任务执行的线程池，多数情况下可用来替代Timer类。
