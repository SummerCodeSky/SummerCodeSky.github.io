数据定义语言DDL用来创建数据库中的各种对象—–表、视图、

数据控制语言DCL

数据控制语言DCL用来授予或回收访问数据库的某种特权，并控制
数据库操纵事务发生的时间及效果，对数据库实行监视等。如：

1) GRANT：授权。

2) ROLLBACK [WORK] TO [SAVEPOINT]：回退到某一点。
回滚—ROLLBACK
回滚命令使数据库状态回到上次最后提交的状态。其格式为：
SQL>ROLLBACK;

3) COMMIT [WORK]：提交。

索引

​ 主要提高从表中检索数据的速度。由于数据是存储在数据库表里面，所以索引是创建在数据表对象上的，有表中的一个字段或者多个字段生成的键组成。这些键存储在数据结构中（b-数或者哈希表）中，通过mysql可以快速有效的查找到与键值相关联的字段。 根据数据结构索引可以分为：b型
树索引，和哈希索引
​ 在mysql中支持的6种索引：，唯一索普通索引引，全文索引，单列
索引，多列索引，空间索引。
​ 创建索引可以提高查询的速度，但过多的创建索引就会占据许多磁盘空间，因此在创建的时候要权衡。
​ 适合建立索引的:
（1）经常被查询的字段id，where语句中出现的字段。
（2）在分组的字段，在group by语句中出现的字段。
（3）存在依赖关系的子表和父表之间的联合查询，主键和外键。
（4）设置唯一完整性约束的字段。
​ 不适合的
（1）查询子句中很少被使用的字段。
（2）拥有多重复的字段。

视图

数据库中的查询有的时候非常复杂，表的链接，子查询。这些查询操作比较的复杂和编写的语句比较的多。有的时候这样的操作还是经常需要的。这样也保证不了正确率。就降低了数据库的实用性。在查询的过程中也不是全都需要查询的字段，那么就要限制程序员的操作字段。

  1. 解决查询的全部查询：select *from 表；把表种所有数据都拿出来，耗时。
  2. 解决sql语句的复用。为了提高数据库的复用性和安全性。就提供了视图。

相关定义

​ 本质就是一个虚拟表，内容和原表内容相似，包括一系列带有名称和行数的数据。但是视图并不在数据库中以存储数据值的形式存在。行和列数据来自定义视图的查询所引用基本表，并且当具体引用视图的时候动态生成。不占用资源.

特点：

1.来自不同表

2.由实际表产生的虚表

3.建立删除视图不影响基表

4.对视图的内容更新（增加、删除、修改）影响基本表

5.视图来自多个表的时候不想允许内容更新（增加、删除、修改）

存储过程

相关定义

​ SQL语句需要先编译然后执行，而存储过程（Stored
Procedure）是一组为了完成特定功能的SQL语句集，经编译后存
储在数据库中，用户通过指定存储过程的名字并给定参数（如果该
存储过程带有参数）来调用执行它。

​ 存储过程是可编程的函数，在数据库中创建并保存，可以由SQL语句和控制结构组成。当想要在不同的应用程序或平台上执行相同的函
数，或者封装特定功能时，存储过程是非常有用的。数据库中的存
储过程可以看做是对编程中面向对象方法的模拟，它允许控制数据
的访问方式。

存储过程的创建

(2)存储过程的输入、输出、输入输出参数如果有多个，那么以“，”隔开。

存储过程和函数的区别：

1）函数理解成一个变量；而存储过程可以返回多个。存储过程的参数可以有IN,OUT,INOUT三种类型，而函数只能有IN类，存储过程声明时不需要返回类型，而函数声明时需要描述返回类型，且函数体中必须包含一个有效的RETURN语句。
2）存储过程一般是作为一个独立的部分来执行（ EXECUTE 语句执行），而函数可以作为查询语句的一个部分来调用（SELECT调用），由于函数可以返回一个表对象，因此它可以在查询语句中位于FROM关键字的后面。 SQL语句中不可用存储过程，而可以使用函数。

事务

相关定义

事务是指逻辑上的一组操作，组成这组操作的各个单元，要不全部成功。

即要么都执行，要么都不执行。双方必须放到一个事务中。

保证了数据操作的安全性。

开启事务（start transaction）

回滚事务（rollback）

提交事务（commit）

最终提交的事务不能再进行回滚操作。

JDBC使用事务

事务的四大特性

原子性(Atomicity)

不可分割的单元，要不全成功，要不全失败。

一致性(consistency)

事务一致性的概念是：事务必须使数据库从一个一致性状态
变换到另外一个一致性状态。

隔离性(Isolation)

多个用户开启事务，相互不干扰，并发执行。

即多个并发事物相互隔离。

事务的隔离级别

​ 多个线程开启各自事务操作数据库中数据时，数据库系统要负责隔离操作，以保证各个线程在获取数据时的准确性。

事务不考虑隔离性可能会引发的问题,如果事务不考虑隔离性，可能
会引发如下问题：

脏读(前一事务未提交的脏数据。)

脏读指一个事务读取了另外一个事务未提交的数据。

不可重复读(重新读取了前一事务已提交的数据)

不可重复读指在一个事务内读取表中的某一行数据，多次读取结果
不同。

不可重复读和脏读的区别是:脏读是读取前一事务未提交的脏数据
，不可重复读是重新读取了前一事务已提交的数据。

虚读（幻读）

虚读(幻读)是指在一个事务内读取到了别的事务插入的数据，导致
前后读取不一致。

事务隔离级别 脏读 不可重复读 幻读
读未提交（read uncommitted） 是 是 是
不可重复读 否 是 是
可重复读（repeatable read） 否 否 是
串行化 否 否 否

隔离级别相关语句：

持久性(Durablity)

事务一旦提交，对事务改变是永久性的，即使数据库故障也不会有任何影响。

总结补充：

1、SQL规范所规定的标准，不同的数据库具体的实现可能会有
些差异

2、mysql中默认事务隔离级别是可重复读时并不会锁住读取到
的行

3、事务隔离级别为读提交时，写数据只会锁住相应的行

4、事务隔离级别为可重复读时，如果有索引（包括主键索引）
的时候，以索引列为条件更新数据，会存在间隙锁间隙锁、行锁、
下一键锁的问题，从而锁住一些行；如果没有索引，更新数据时会
锁住整张表。

5、事务隔离级别为串行化时，写、修改、删除数据都会锁住整
张表

6、隔离级别越高，越能保证数据的完整性和一致性，但是对并
发性能的影响也越大，鱼和熊掌不可兼得啊。对于多数应用程序，
可以优先考虑把数据库系统的隔离级别设为Read Committed，它
能够避免脏读取，而且具有较好的并发性能。尽管它会导致不可重
复读、幻读这些并发问题，在可能出现这类问题的个别场合，可以
由应用程序采用悲观锁或乐观锁来控制。
