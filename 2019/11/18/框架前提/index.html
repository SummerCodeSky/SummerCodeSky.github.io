Schema文档后缀名是.xsd

Schema 是一个单拿出来的xml；把我们需要的标签都写在xml文件中； 在使用的时候，直接去加载你的schema文件加载过来；

数据存储

1.开始数据存储在定义出来的变量；

2.能够存储多个变量时候，使用到数组；

3.在数组中存储对象；就是对象数组；

4.io流之后直接可以数据存储在文件，（对象序列化—就是把对象转换之后存储在文件上；对象反序列化—就是从文件上有把对象读取出来；）

5.xml文件，直接可以把数据存储在你的xml文件上；通过很多标签来进行存储；对象直接可以对应存储在xml文件上；

6.数据库—最终的都是把数据存储在数据库中， 表的方式来存储；表和表之间有了关系；—-关系型数据库；

使用javaee部分的内容：

1.servlet：进行web开发必须使用的一个技术；他里主要是你的 ：

请求和响应

2.servlet优先于你的jsp存在；servlet是一个纯的java代码；他的主要作用后端处理；

JSP

​ jsp是一个混合的一个页面（可以写html—页面展示； css提供给你的html样式； js来对你的页面进行具体的一些操作；前端验证； java代码）；jsp主要的作用是用来进行页面展示用；

​ jsp要运行的时候： 必须通过tomcat，jboos 等等；容器；xxx. jsp—通过容器进行编译会生成一个xxx.java文件；这个文件是通过容器进行编译之后得到的文件； 这个文件通过java里边的编译器—xxx.java 编译—–生成一个xxx.class 文件； 运行的时候我们的虚拟机就会调用你这个xxx.class文件；——jsp的运行过程；

​ servlet—他是直接编译成class文件；他要显示数据的时候是通过我们的文件流的方式来打到你的页面上；

JSP-9大内置对象

jsp—9大内置对象；—-内置对象就是不需要创建对象，已经有了这个对象；

JSP三大指令

jsp三的指令： include（包含）；page（是用来设置页面属性；）taglib（引入外部标签库—jstl标签库的时候使用）

经常使用内容：

1.请求: request

2.响应： response

3.回话：session—服务器端存储

4.cookie—浏览器端存储

5.请求转发：forward他是一次请求，可以带着你的数据到你的新的页面；

6.重定向：他是两次请求，不能带着数据走；**sendRedirect 地址栏中的url 是要发生变化的；

7.页面请求的时候：get请求，post请求；

Get和post的区别：

1.get请求是明文的是在url后边会加上你的数据； post他是隐藏你的数据；url是看不见数据的；

get参数通过url传递，post放在request body中。

get请求在url中传递的参数是有长度限制的，而post没有。

get比post更不安全，因为参数直接暴露在url中，所以不能用来传递敏感信息。

get请求只能进行url编码，而post支持多种编码方式

get请求会浏览器主动cache，而post支持多种编码方式。

get请求参数会被完整保留在浏览历史记录里，而post中的参数不会被保留。

乱码问题

get请求乱码； post请求乱码等；

监听器、过滤器

Servlet里边的过滤器（可以过滤请求数据；）；监听器（在servlet里边一个有8个监听器； 可以监听你的请求，监听你的session，监听你的上下文–context）只要这些这些被监听内容发生变化；就会触发你的监听器；

Servlet 监听器，过滤器；
